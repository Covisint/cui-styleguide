1.3 Colors

DESCRIPTION:

A color palette for the Covisint Next Gen UI Framework. Values are available as Sass variables. Variable names are indicated inside each color panel.

1.3.1 Primary Colors

EXAMPLE:

$cui-primary-colors--medium-light #429DD1
$cui-primary-colors--medium #4582B4
$cui-primary-colors--medium-dark #165E9B
$cui-primary-colors--dark #0A2741

1.9 Texts

DESCRIPTION:

Styles for text. Please refer to _vars.scss for variable values.

1.9.1 Headers

DESCRIPTION:

Header styling is applied through the use of classes. Our recommended method for applying h1 styling is to @extend the %h1 placeholder selector in a class's CSS style declaration block and apply that class to your element.

EXAMPLE:

H1 Heading

H2 Heading

H3 Heading

H4 Heading

H5 Heading
H6 Heading
CODE:
<h1 class="h1">H1 Heading</h1>
<h2 class="h2">H2 Heading</h2>
<h3 class="h3">H3 Heading</h3>
<h4 class="h4">H4 Heading</h4>
<h5 class="h5">H5 Heading</h5>
<h6 class="h6">H6 Heading</h6>
<div class="cui-styleguide__text-section">
</div>

1.9.4 Ordered Lists

DESCRIPTION:

Browser styles are removed from lists in order to separate visual presentation from semantic meaning. If you want to restore the default styling, @extend the placeholder selector %cui-ol--default in your list's CSS style declaration block.

EXAMPLE:
Base styles
  1. Ordered List
  2. Ordered List
  3. Ordered List
.cui-ol--default
restores browser styling
  1. Ordered List
  2. Ordered List
  3. Ordered List
CODE:
<ol class="[modifier class]">
  <li>Ordered List</li>
  <li>Ordered List</li>
  <li>Ordered List</li>
</ol>
